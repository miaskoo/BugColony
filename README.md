# BugColony

Симуляция колонии жуков на Unity 6 (6000.3.10f1).

Жуки-рабочие собирают еду, делятся и с некоторой вероятностью мутируют в хищников. Хищники охотятся на рабочих, имеют ограниченное время жизни и неуязвимость после появления. Если все жуки погибли — колония автоматически возрождается.

## Архитектура

### Структура папок

```
Assets/Scripts/
├── Bug/            — модели жуков, конфиги, визуализация
├── Core/           — базовый класс SceneEntity
├── Environment/    — объекты среды (FoodObject, GroundController)
├── Infrastructure/ — Zenject-инсталлеры, сигналы, ColonyController
├── Spawning/       — пул объектов, спаунеры жуков и еды
├── Strategy/       — паттерн «Стратегия» для поведения жуков
└── UI/             — UI-контроллер счётчика погибших жуков
```

### Ключевые компоненты

| Компонент | Описание |
|---|---|
| **SceneEntity** | Базовый класс для всех объектов на сцене (жуки, еда). Управляет активацией/деактивацией GameObject. |
| **BugObject** | Конкретный жук. Хранит тип (`Worker`/`Predator`), очки «желудка», оставшееся время жизни. Делегирует поведение стратегии. |
| **BugConfig** | ScriptableObject с общими параметрами жука (скорость, еда, деление, неуязвимость). Наследники: `WorkerBugConfig`, `PredatorBugConfig`. |
| **IBugStrategy** | Интерфейс стратегии поведения. Определяет движение, питание, мутацию. |
| **BugStrategyWorker** | Рабочий: ест еду, при наборе очков делится; может мутировать в хищника. |
| **BugStrategyPredator** | Хищник: ест рабочих, живёт ограниченное время, при делении порождает хищника. |
| **BugStrategyController** | Реестр стратегий. Возвращает нужную стратегию по `BugType`. |
| **ColonyController** | Главный контроллер сцены. Спаунит первого рабочего, периодически создаёт еду, возрождает колонию при вымирании. |
| **ObjectPool\<T\>** | Универсальный пул объектов с авторасширением. |
| **BugSpawner / FoodSpawner** | Спаунеры, использующие пул. `BugSpawner` хранит маппинг `BugType → BugConfig`. |
| **GroundController** | Находит свободную позицию на поверхности земли с учётом коллизий. |
| **DeadBugUIController** | Подписывается на сигнал `BugKilledSignal` и отображает счётчики убитых рабочих и хищников. |

## Настройка параметров

Все параметры жуков настраиваются через ScriptableObject-конфиги, остальное в объектах на сцене например в `ColonyController` или `GroundController`.

### Конфиги жуков

Расположены в `Assets/Resources/Configs/`.


### Pipeline добавления нового типа жука

1. Добавьте значение в `BugType` enum.
2. Создайте наследника `BugConfig` (ScriptableObject) с нужными параметрами.
3. Создайте наследника `BugStrategyBase<TConfig>`, реализовав `OnBugActivated`, `TryMutate`, и при необходимости переопределив `CreateMoveAction`.
4. Зарегистрируйте новую стратегию в `BugSceneInstaller`.
5. Создайте конфиг-ассет в `Assets/Resources/Configs/` и назначьте его в `BugSpawner` на сцене.
